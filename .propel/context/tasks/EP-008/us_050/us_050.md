# User Story - US_050

## Story ID

- ID: US_050

## Story Title

- AI-Human Agreement Rate Tracking

## Description

- As an admin, I want the system to calculate and display the AI-human agreement rate for medical coding with daily metrics, so that I can monitor AI coding accuracy and identify when retraining or model adjustments are needed.

## Acceptance Criteria

1. **Given** staff verify or override AI-suggested codes, **When** the agreement rate is calculated, **Then** the system computes the percentage of AI codes approved without override (target >98%).
2. **Given** the agreement rate is calculated daily, **When** an admin views the metrics dashboard, **Then** the daily and rolling 30-day agreement rate is displayed.
3. **Given** a coding discrepancy exists (multiple codes for single diagnosis/procedure), **When** the system detects it, **Then** the discrepancy is flagged with a breakdown of AI suggestion vs. staff selection.
4. **Given** the agreement rate drops below 98%, **When** the daily calculation runs, **Then** the system generates an alert for admin review with a summary of disagreement patterns.

## Edge Cases

- What happens when insufficient data exists for statistical significance? System displays "Not enough data" with a minimum threshold indicator (e.g., requires 50+ verified codes per period).
- How does the system handle agreement calculation when codes are partially overridden (e.g., correct code but wrong modifier)? Partial overrides count as disagreements in the agreement rate.
