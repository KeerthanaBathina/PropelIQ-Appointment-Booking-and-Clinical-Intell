# User Story - US_049

## Story ID

- ID: US_049

## Story Title

- Staff Code Verification and Override Workflow

## Description

- As a staff member, I want to review AI-suggested codes for verification and approval, override them with manual selections when needed, and have all changes logged with justification, so that coding accuracy is ensured with full audit trails.

## Acceptance Criteria

1. **Given** AI has generated ICD-10 and CPT codes, **When** the staff member views the medical coding screen, **Then** codes are presented in a verification queue with AI justification, confidence score, and approve/override actions.
2. **Given** a staff member approves an AI-suggested code, **When** they click "Approve," **Then** the code status changes to "verified" with staff attribution and timestamp in the audit trail.
3. **Given** a staff member disagrees with an AI suggestion, **When** they select "Override," **Then** the system presents a code search interface and requires the staff member to enter a justification for the override.
4. **Given** a code change is made (approval or override), **When** the change is saved, **Then** an immutable audit log entry records the old code, new code, justification, user, and timestamp.

## Edge Cases

- What happens when a staff member tries to approve a code that has been deprecated in a recent library update? System blocks approval and shows the deprecated notice with suggested replacement codes.
- How does the system handle partial verification (some codes approved, some pending)? Patient coding status shows "partially verified" with a progress indicator (e.g., 3/5 codes verified).
