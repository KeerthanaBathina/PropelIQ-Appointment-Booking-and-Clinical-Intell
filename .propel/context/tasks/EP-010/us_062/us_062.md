# User Story - US_062

## Story ID

- ID: US_062

## Story Title

- Patient Search and Profile View

## Description

- As a staff member, I want to search for patients by name, date of birth, or phone number, and view their complete profile including appointments, intake data, and clinical documents, so that I have comprehensive patient context for clinical and administrative decisions.

## Acceptance Criteria

1. **Given** the staff member opens the patient search, **When** they enter a name, DOB, or phone number, **Then** the system returns matching results within 1 second with relevance ranking.
2. **Given** search results are displayed, **When** the staff member selects a patient, **Then** the system navigates to the complete patient profile view.
3. **Given** the patient profile is displayed, **When** it loads, **Then** it shows sections for demographics, appointment history, intake data, uploaded documents, extracted data, and medical codes.
4. **Given** the staff member views the patient profile, **When** they click on any section, **Then** the section expands to show detailed information with links to source screens.

## Edge Cases

- What happens when the search returns no results? System displays "No patients found" with a suggestion to verify search criteria or create a new patient record.
- How does the system handle partial name searches? System supports partial matching (e.g., "Joh" matches "John," "Johnston") with highlighting of matched characters.
