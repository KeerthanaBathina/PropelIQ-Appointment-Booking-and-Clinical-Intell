# User Story - US_061

## Story ID

- ID: US_061

## Story Title

- Staff Account Management

## Description

- As an admin, I want to create and manage staff user accounts, including deactivation without deleting historical data, so that staff access is controlled while preserving audit and operational history.

## Acceptance Criteria

1. **Given** the admin opens user management, **When** they create a new staff account, **Then** the system provisions the account with role assignment (Staff or Admin), temporary password, and email invitation.
2. **Given** the admin views staff accounts, **When** the list loads, **Then** it displays name, email, role, status (active/deactivated), last login, and creation date.
3. **Given** the admin deactivates a staff account, **When** they confirm deactivation, **Then** the account is disabled (cannot log in) but all historical data (audit logs, actions, verifications) is preserved.
4. **Given** a staff account is deactivated, **When** an admin reactivates it, **Then** the account is restored with previous role and permissions intact.

## Edge Cases

- What happens when an admin tries to deactivate their own account? System prevents self-deactivation with an error "Cannot deactivate your own account."
- How does the system handle deactivation of the last admin account? System prevents it with an error "At least one active admin account required."
