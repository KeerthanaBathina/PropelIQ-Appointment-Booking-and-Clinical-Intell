# User Story - US_058

## Story ID

- ID: US_058

## Story Title

- Admin Dashboard with System Metrics

## Description

- As an admin, I want a dashboard displaying system metrics, user management, and configuration settings, so that I can monitor system health and manage the platform from a single interface.

## Acceptance Criteria

1. **Given** the admin logs in, **When** the admin dashboard loads, **Then** it displays system metrics (active users, daily appointments, no-show rate, AI agreement rate, uptime percentage).
2. **Given** the metrics section is displayed, **When** the admin views trends, **Then** the system shows rolling 7-day and 30-day trend charts for key metrics.
3. **Given** the admin dashboard is loaded, **When** the admin navigates to user management, **Then** they see a list of all staff and admin accounts with status, last login, and role.
4. **Given** the admin clicks configuration, **When** the configuration panel opens, **Then** it provides tabs for appointment templates, business hours, notification templates, and risk thresholds.

## Edge Cases

- What happens when system metrics data is temporarily unavailable? Dashboard shows cached values with a "Data as of [timestamp]" indicator.
- How does the system handle the admin dashboard on mobile screens? Dashboard collapses metrics into collapsible cards and stacks configuration sections vertically.
