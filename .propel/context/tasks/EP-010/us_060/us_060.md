# User Story - US_060

## Story ID

- ID: US_060

## Story Title

- Notification and Risk Configuration

## Description

- As an admin, I want to configure notification templates for email and SMS messages, and adjust no-show risk thresholds and scoring parameters, so that communications and risk management align with clinic policies.

## Acceptance Criteria

1. **Given** the admin opens notification template configuration, **When** they select a template (booking confirmation, 24h reminder, 2h reminder), **Then** they can edit subject line, body text, and variable placeholders (patient name, date, time, provider).
2. **Given** the admin edits a notification template, **When** they save changes, **Then** all future notifications use the updated template without affecting already-sent messages.
3. **Given** the admin opens risk configuration, **When** they adjust the no-show risk threshold, **Then** the system recalculates risk scoring display using the new threshold values.
4. **Given** the admin modifies scoring parameters, **When** they save changes, **Then** the system logs the parameter change with admin attribution and timestamp.

## Edge Cases

- What happens when a template edit introduces invalid variable placeholders? System validates template syntax on save and rejects templates with unrecognized variables.
- How does the system handle risk threshold changes affecting active appointments? Risk scores for existing appointments are recalculated in the next batch run (not retroactively applied immediately).
