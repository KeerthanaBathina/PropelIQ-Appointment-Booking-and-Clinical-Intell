# User Story - US_056

## Story ID

- ID: US_056

## Story Title

- Queue Filtering and History

## Description

- As a staff member, I want to filter the queue by provider, appointment type, and arrival status, and access queue history for operational analytics, so that I can focus on relevant patients and analyze flow patterns.

## Acceptance Criteria

1. **Given** the queue dashboard is displayed, **When** the staff member selects a provider filter, **Then** the queue shows only entries for the selected provider.
2. **Given** the queue dashboard is displayed, **When** the staff member applies multiple filters (provider + appointment type + status), **Then** filters are combined with AND logic and results update instantly.
3. **Given** the staff member navigates to queue history, **When** they select a date range, **Then** the system displays historical queue data including average wait times, no-show counts, and patient throughput.
4. **Given** queue history data is available, **When** the staff member requests an export, **Then** the system generates a CSV report with queue metrics for the selected period.

## Edge Cases

- What happens when no queue entries match the applied filters? System displays an empty state message "No patients match the selected filters" with a clear-all-filters button.
- How does the system handle queue history for dates before system deployment? System shows "No data available for the selected period" with available date range indicated.
