# User Story - US_053

## Story ID

- ID: US_053

## Story Title

- Real-Time Arrival Queue Dashboard

## Description

- As a staff member, I want a real-time arrival queue dashboard sorted by appointment time and priority with sub-second response time, so that I can efficiently manage patient flow throughout the day.

## Acceptance Criteria

1. **Given** the staff member opens the arrival queue, **When** the dashboard loads, **Then** it displays all arrived patients sorted by appointment time and priority with sub-second response time.
2. **Given** the queue is displayed, **When** a patient's status changes (new arrival, no-show, cancelled), **Then** the dashboard refreshes within 5 seconds without requiring manual reload.
3. **Given** the queue dashboard is open, **When** the staff member views it, **Then** each entry shows patient name, appointment time, provider, arrival time, current wait time, and status badge (color-coded).
4. **Given** the queue contains entries, **When** the staff member calculates average wait time, **Then** the system displays the average wait time for all currently waiting patients.

## Edge Cases

- What happens when the dashboard has 100+ queue entries? System paginates with 25 entries per page and maintains sort order across pages.
- How does the system handle network latency for real-time updates? System uses optimistic UI updates with a "Last updated" timestamp; manual refresh button available.
