# User Story - US_025

## Story ID

- ID: US_025

## Story Title

- Calendar Sync via iCal Format

## Description

- As a patient, I want to sync my appointments to Google Calendar or Outlook, so that my appointments appear alongside my personal schedule.

## Acceptance Criteria

1. **Given** I have a confirmed appointment, **When** I click "Add to Calendar", **Then** the system generates an iCal (.ics) file with appointment details (date, time, provider, location).

2. **Given** the iCal file is downloaded, **When** I open it, **Then** Google Calendar and Outlook correctly import the event with proper timezone handling.

3. **Given** I reschedule an appointment, **When** a new iCal is generated, **Then** it includes the UID of the original event to update (not duplicate) the calendar entry.

## Edge Cases

- What happens when the patient's calendar app doesn't support iCal? The .ics file is a universal standard; fallback is manual entry with displayed appointment details.
- How does the system handle timezone differences? iCal uses VTIMEZONE component with the clinic's timezone; display converts to user's local time.
