# User Story - US_024

## Story ID

- ID: US_024

## Story Title

- Appointment History & Status Tracking

## Description

- As a patient, I want to view my appointment history with status tracking, so that I can review past visits and upcoming appointments in one place.

## Acceptance Criteria

1. **Given** I navigate to appointment history, **When** the page loads, **Then** all my appointments are displayed in a paginated table sorted by date (newest first) with status badges (scheduled, completed, cancelled, no-show).

2. **Given** the history table is displayed, **When** I click a column header (date), **Then** the table sorts by that column in ascending/descending order.

3. **Given** I have 50+ appointments, **When** the table loads, **Then** pagination shows 10 items per page with Previous/Next navigation.

4. **Given** appointments exist, **When** the data is fetched, **Then** each row shows date, time, provider, type, and status badge with consistent status colors per UXR-401.

## Edge Cases

- What happens when a patient has no appointment history? System displays "No appointments found. Book your first appointment!" with link to SCR-006.
- How does the system handle cancelled appointments in history? Displayed with "Cancelled" grey badge and reduced opacity.
