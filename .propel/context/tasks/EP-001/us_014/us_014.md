# User Story - US_014

## Story ID

- ID: US_014

## Story Title

- Secure Session Management

## Description

- As a user, I want my session to automatically expire after 15 minutes of inactivity, so that my account is protected from unauthorized access on unattended devices.

## Acceptance Criteria

1. **Given** I am logged in, **When** I am inactive for 15 minutes, **Then** the system invalidates my session and redirects to login with message "Session expired due to inactivity."

2. **Given** I am actively using the system, **When** I make a request within the 15-minute window, **Then** the session timer resets and I remain authenticated.

3. **Given** I am logged in on one device, **When** I attempt to log in on a second device, **Then** the system rejects the second login with "Another active session exists. Please logout first."

4. **Given** a session expiry warning is configured, **When** 2 minutes remain before expiry, **Then** a modal countdown appears with an "Extend Session" button.

## Edge Cases

- What happens when the server restarts mid-session? Redis-backed session storage preserves session state across server restarts.
- How does the system handle simultaneous requests from the same session? Last-activity timestamp updates atomically; no race condition on session extension.
