# User Story - US_012

## Story ID

- ID: US_012

## Story Title

- Patient Registration with Email Verification

## Description

- As a new patient, I want to create an account with email and password including email verification, so that I can securely access the platform's appointment and intake features.

## Acceptance Criteria

1. **Given** I am on the registration page, **When** I submit valid name, email, phone, DOB, and password, **Then** an account is created with status "pending verification" and a verification email is sent within 2 minutes.

2. **Given** I received a verification email, **When** I click the verification link, **Then** my account status changes to "active" and I am redirected to the login page with a success message.

3. **Given** the verification link has expired (1-hour expiry), **When** I click it, **Then** the system displays "Link expired" and offers to resend a new verification email.

4. **Given** I submit a registration with an already-registered email, **When** the form is submitted, **Then** the system displays "An account with this email already exists" without revealing whether the email is verified.

5. **Given** the password field is filled, **When** the password does not meet complexity requirements (8+ chars, 1 uppercase, 1 number, 1 special char), **Then** inline validation displays specific missing criteria within 200ms.

## Edge Cases

- What happens when a user requests multiple verification emails? Rate limiting: max 3 resend requests per 5 minutes per email address.
- How does the system handle registration with a disposable email domain? Allow in Phase 1; consider domain allowlist in Phase 2.
