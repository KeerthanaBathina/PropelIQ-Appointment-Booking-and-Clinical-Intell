# User Story - US_015

## Story ID

- ID: US_015

## Story Title

- Password Reset via Email Token

## Description

- As a user who forgot my password, I want to reset it via a secure email token, so that I can regain access to my account without contacting support.

## Acceptance Criteria

1. **Given** I am on the login page and click "Forgot Password", **When** I enter my registered email address, **Then** the system sends a password reset link within 2 minutes.

2. **Given** I receive the reset email, **When** I click the reset link within 1 hour, **Then** I am directed to a secure page where I can set a new compliant password.

3. **Given** the reset link has expired (1-hour window), **When** I click it, **Then** the system displays "Link expired" and offers to resend a new reset link.

4. **Given** I successfully reset my password, **When** I try to log in with the old password, **Then** the system rejects it and only accepts the new password.

## Edge Cases

- What happens when a non-registered email is submitted? System displays the same "If an account exists, a reset link was sent" message to prevent email enumeration.
- What happens when a user requests multiple resets? Only the latest token is valid; previous tokens are invalidated.
