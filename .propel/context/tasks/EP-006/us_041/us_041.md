# User Story - US_041

## Story ID

- ID: US_041

## Story Title

- Confidence Scoring and Low-Confidence Flagging

## Description

- As a staff member, I want each extracted data point to display a confidence score with visual indicators and data below 80% confidence flagged for manual verification, so that I can prioritize my review efforts on uncertain extractions.

## Acceptance Criteria

1. **Given** AI extraction produces data points, **When** each data point is stored, **Then** a confidence score (0.00-1.00) is calculated and saved with the ExtractedData record.
2. **Given** an extracted data point has confidence below 0.80, **When** the results are displayed, **Then** the data point is visually flagged with an amber/red indicator and marked for mandatory manual verification.
3. **Given** a staff member views the extraction results, **When** they see confidence scores, **Then** scores are color-coded: green (≥0.90), amber (0.80-0.89), red (<0.80).
4. **Given** a data point is flagged for verification, **When** the staff member confirms or corrects it, **Then** the verification status updates to "verified" with staff attribution and timestamp.

## Edge Cases

- What happens when the AI cannot assign a confidence score? System defaults to 0.00 and flags the data point as "confidence-unavailable" for mandatory review.
- How are batch verifications handled? Staff can select multiple flagged items and mark them as "verified" in bulk with a single confirmation.
