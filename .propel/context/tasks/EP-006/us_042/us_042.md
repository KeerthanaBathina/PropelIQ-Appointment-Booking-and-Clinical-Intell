# User Story - US_042

## Story ID

- ID: US_042

## Story Title

- Document Preview and Re-Upload Workflow

## Description

- As a staff member, I want to preview uploaded documents with highlighted extraction regions and re-upload documents when initial extraction quality is low, so that I can verify AI accuracy and correct poor-quality uploads.

## Acceptance Criteria

1. **Given** a document has been parsed, **When** the staff member opens the document preview, **Then** the system displays the original document with highlighted regions indicating where data was extracted.
2. **Given** a staff member determines extraction quality is insufficient, **When** they click "Re-Upload," **Then** the system allows a new version upload that replaces the previous document and triggers re-processing.
3. **Given** a document is re-uploaded, **When** parsing completes, **Then** previous extracted data is archived (not deleted) and new extraction results replace the active data.
4. **Given** a staff member views a document preview, **When** hovering over a highlighted region, **Then** a tooltip shows the extracted data value and confidence score for that region.

## Edge Cases

- What happens when the document format doesn't support region highlighting (e.g., TXT)? System shows the full document text with extracted values annotated inline.
- How does re-upload affect downstream consolidated profiles? System triggers re-consolidation of the patient's 360-degree profile with the updated extraction data.
