# User Story - US_070

## Story ID

- ID: US_070

## Story Title

- Token Budget Enforcement and Circuit Breaker

## Description

- As a system administrator, I want per-request token budget enforcement and a circuit breaker that opens after 5 consecutive failures with 30-second retry, so that AI costs are controlled and cascading failures are prevented.

## Acceptance Criteria

1. **Given** the AI Gateway processes requests, **When** a request's token count exceeds the configured budget, **Then** the request is rejected with a "token budget exceeded" error before being sent to the provider.
2. **Given** the circuit breaker monitors provider health, **When** 5 consecutive failures occur, **Then** the circuit breaker opens and all new requests are immediately routed to the fallback provider.
3. **Given** the circuit breaker is open, **When** 30 seconds elapse, **Then** the circuit enters half-open state and allows one test request to determine if the provider has recovered.
4. **Given** the half-open test succeeds, **When** the response is received, **Then** the circuit breaker closes and normal routing resumes.

## Edge Cases

- What happens when both providers trigger circuit breakers simultaneously? System enters degraded mode, returns "AI unavailable" to all callers, and notifies admins via alert.
- How does the system handle circuit breaker state across multiple server instances? Circuit breaker state is shared via Redis to ensure consistent behavior across all instances.
