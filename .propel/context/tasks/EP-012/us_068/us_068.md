# User Story - US_068

## Story ID

- ID: US_068

## Story Title

- OpenAI GPT-4o-mini Primary Provider Integration

## Description

- As a developer, I want OpenAI GPT-4o-mini integrated as the primary AI provider with per-request token budget enforcement, so that all AI features use the primary model with controlled costs.

## Acceptance Criteria

1. **Given** the AI Gateway is configured, **When** an AI request is routed, **Then** OpenAI GPT-4o-mini is used as the primary provider by default.
2. **Given** a document parsing request is sent, **When** token budget is enforced, **Then** the request is limited to 4K input tokens and 1K output tokens per the AIR-O01 specification.
3. **Given** an intake conversational request is sent, **When** token budget is enforced, **Then** the request is limited to 500 input tokens and 200 output tokens per the AIR-O02 specification.
4. **Given** a medical coding request is sent, **When** token budget is enforced, **Then** the request is limited to 2K input tokens and 500 output tokens per the AIR-O03 specification.

## Edge Cases

- What happens when a request exceeds the token budget? System truncates the input to fit within budget, logs the truncation, and includes a "truncated" flag in the response.
- How does the system handle OpenAI API rate limits? System implements exponential backoff (max 3 retries) and queues excess requests rather than failing immediately.
