# User Story - US_081

## Story ID

- ID: US_081

## Story Title

- Performance Optimization for Core Operations

## Description

- As a performance engineer, I want booking response time optimized to <2s, document parsing to <30s, and medical coding to <5s at P95, so that the system meets performance SLAs under normal and peak load.

## Acceptance Criteria

1. **Given** a patient submits a booking request, **When** the request is processed, **Then** the end-to-end response completes within 2 seconds at P95.
2. **Given** a clinical document is uploaded, **When** AI parsing runs, **Then** extraction completes within 30 seconds at P95.
3. **Given** AI medical coding is triggered, **When** codes are generated, **Then** the coding response completes within 5 seconds per diagnosis/procedure at P95.
4. **Given** performance is monitored, **When** any metric exceeds its SLA threshold, **Then** the system generates a performance alert with the affected operation, P95 value, and trend.

## Edge Cases

- What happens when performance degrades under peak load? System prioritizes booking requests over parsing and coding via request priority queues.
- How does the system identify performance bottlenecks? Application Performance Monitoring (APM) traces all operations with span-level timing for database, cache, and external API calls.
