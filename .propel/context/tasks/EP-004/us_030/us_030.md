# User Story - US_030

## Story ID

- ID: US_030

## Story Title

- Intake Auto-Save & Field Collection

## Description

- As a patient, I want my intake progress auto-saved every 30 seconds, so that I do not lose my data if my session is interrupted.

## Acceptance Criteria

1. **Given** I am filling intake (AI or manual), **When** 30 seconds elapse since the last change, **Then** the system saves current progress to the server and displays "Auto-saved" indicator.

2. **Given** my session is interrupted (timeout, network loss), **When** I return to the intake, **Then** the system restores the last auto-saved state with all fields populated.

3. **Given** the auto-save triggers, **When** the save completes, **Then** the auto-save indicator briefly appears and fades, confirming the save without disrupting the user flow.

## Edge Cases

- What happens when auto-save fails due to network error? System retries once after 5 seconds; if still failed, displays "Save failed - your responses are cached locally."
- How does the system handle rapid changes within 30 seconds? Only the state at the 30-second mark is saved; intermediate states are not persisted.
