# User Story - US_028

## Story ID

- ID: US_028

## Story Title

- Manual Form-Based Intake

## Description

- As a patient, I want to complete my intake using a traditional web form, so that I have an alternative to the AI chatbot for providing my medical information.

## Acceptance Criteria

1. **Given** I select "Manual Form" from intake options, **When** the form loads, **Then** it displays sections for Personal Information, Medical History, and Insurance with all mandatory and optional fields.

2. **Given** AI intake has pre-filled some data, **When** the manual form loads, **Then** the pre-filled fields display the AI-collected data with visual indication of pre-filled status.

3. **Given** I am filling out the form, **When** I leave a mandatory field empty and attempt to submit, **Then** inline validation highlights the missing field within 200ms with a descriptive error.

4. **Given** I complete and submit the form, **When** submission succeeds, **Then** intake status is marked "completed" and staff is notified for review.

## Edge Cases

- What happens when I navigate away from the form mid-completion? Auto-save preserves progress; returning loads the last saved state.
- How does the system handle browser back button during form submission? Form prevents double-submit; back navigates without data loss.
