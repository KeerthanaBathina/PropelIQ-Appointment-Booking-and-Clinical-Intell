# User Story - US_027

## Story ID

- ID: US_027

## Story Title

- AI Conversational Intake Interface

## Description

- As a patient, I want to complete my intake using an AI conversational chatbot that asks questions in natural language, so that I can provide my medical information in a comfortable, guided experience.

## Acceptance Criteria

1. **Given** I select "AI-Assisted Intake" from my dashboard, **When** the chat interface loads, **Then** the AI greets me and explains the intake process with a progress indicator.

2. **Given** the AI asks a question, **When** I provide a response, **Then** the AI validates my answer, asks clarifying follow-ups if needed, and moves to the next question within 1 second.

3. **Given** mandatory fields are required, **When** the AI collects my information, **Then** it asks for full name, DOB, contact info, and emergency contact before marking mandatory collection complete.

4. **Given** the intake session is active, **When** the AI displays a summary, **Then** I can review all collected information and correct any errors before submission.

5. **Given** the conversation progresses, **When** I look at the progress bar, **Then** it accurately reflects the number of fields collected out of total required (e.g., "4/8 fields").

## Edge Cases

- What happens when I provide ambiguous medical terminology? AI uses RAG context to ground medical terms and asks for clarification with examples.
- How does the system handle session timeout during intake? Last auto-saved state is restored when I return; AI resumes from the last completed question.
