# User Story - US_034

## Story ID

- ID: US_034

## Story Title

- Booking Confirmation Notifications (Email + SMS + PDF)

## Description

- As a patient, I want to receive an immediate booking confirmation via email and SMS with a PDF attachment containing a QR code, so that I have reliable proof of my appointment.

## Acceptance Criteria

1. **Given** a patient successfully books an appointment, **When** the booking is confirmed, **Then** the system sends a confirmation email and SMS within 30 seconds.
2. **Given** the confirmation is triggered, **When** the email is composed, **Then** it includes a PDF attachment with appointment details and a scannable QR code for check-in.
3. **Given** a booking confirmation is sent, **When** the patient views the email, **Then** it contains a cancellation link that pre-fills the cancellation workflow.
4. **Given** the patient has opted out of SMS, **When** a booking confirmation is triggered, **Then** only email (with PDF) is sent and SMS is skipped with status logged as "opted-out."

## Edge Cases

- What happens when PDF generation fails? System sends email without PDF, logs generation failure, and queues a retry for PDF generation.
- How does the system handle simultaneous booking and cancellation within seconds? Cancellation takes priority; confirmation email includes a "cancelled" note if processed after cancellation.
