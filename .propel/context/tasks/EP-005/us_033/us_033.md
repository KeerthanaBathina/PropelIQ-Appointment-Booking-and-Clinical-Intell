# User Story - US_033

## Story ID

- ID: US_033

## Story Title

- SMS Service Integration (Twilio)

## Description

- As a system, I want Twilio API integration for SMS delivery, so that patients receive appointment reminders and notifications via text message.

## Acceptance Criteria

1. **Given** Twilio is configured, **When** an SMS is triggered, **Then** it is sent to the patient's phone number within 30 seconds.
2. **Given** the patient opts out of SMS, **When** an SMS reminder is scheduled, **Then** the system skips SMS and logs "opted-out" status.
3. **Given** SMS delivery fails, **When** the failure is detected, **Then** the system retries up to 3 times with exponential backoff and logs each attempt.
4. **Given** delivery succeeds, **When** Twilio confirms delivery, **Then** status is logged as "sent" in NotificationLog.

## Edge Cases

- What happens when Twilio trial credits are exhausted? System disables SMS, logs alert, and continues with email-only notifications.
- How does the system handle international phone numbers? Phase 1 supports US numbers only (+1); international numbers are rejected with validation error.
