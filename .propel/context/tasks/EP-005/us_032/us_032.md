# User Story - US_032

## Story ID

- ID: US_032

## Story Title

- Email Service Integration (SMTP)

## Description

- As a system, I want SMTP integration with SendGrid or Gmail for email delivery, so that appointment confirmations and reminders reach patients reliably.

## Acceptance Criteria

1. **Given** SMTP is configured, **When** an email is triggered, **Then** it is sent via the configured SMTP provider within 30 seconds.

2. **Given** email delivery is attempted, **When** the send succeeds, **Then** delivery status is logged as "sent" with timestamp in NotificationLog.

3. **Given** the SMTP provider is unreachable, **When** delivery fails, **Then** the system retries up to 3 times with exponential backoff (1s, 2s, 4s) and logs each attempt.

4. **Given** an email template is used, **When** the email is composed, **Then** variable placeholders (patient name, date, time, provider) are replaced with actual values.

## Edge Cases

- What happens when SendGrid free tier limit (100/day) is exceeded? System falls back to Gmail SMTP and alerts admin.
- How does the system handle invalid recipient email? Logs "bounced" status and flags the patient record for contact update.
