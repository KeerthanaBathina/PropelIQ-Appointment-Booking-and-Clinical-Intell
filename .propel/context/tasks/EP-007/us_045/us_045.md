# User Story - US_045

## Story ID

- ID: US_045

## Story Title

- Staff Conflict Resolution with Side-by-Side View

## Description

- As a staff member, I want to resolve flagged clinical data conflicts using a side-by-side comparison view showing conflicting data from both source documents, so that I can make informed decisions about which data is accurate.

## Acceptance Criteria

1. **Given** conflicts are flagged for a patient, **When** the staff member opens the conflict resolution view, **Then** the system displays conflicting data side-by-side with source document citations.
2. **Given** the staff member reviews a conflict, **When** they select the correct data value, **Then** the chosen value is saved to the consolidated profile and the conflict is marked "resolved" with staff attribution.
3. **Given** a conflict involves multiple data sources, **When** the comparison view loads, **Then** each source is displayed with document name, upload date, and confidence score.
4. **Given** a staff member resolves all conflicts for a patient, **When** the last conflict is resolved, **Then** the profile status updates to "verified" and an audit log entry is created.

## Edge Cases

- What happens when a staff member partially resolves conflicts and navigates away? Resolution progress is saved; unresolved conflicts remain flagged for the next session.
- How does the system handle conflicts where both values may be correct (e.g., dosage change)? Staff can select "Both Valid — Different Dates" which preserves both entries with distinct date attribution.
