# User Story - US_046

## Story ID

- ID: US_046

## Story Title

- Manual Fallback and Date Validation for Consolidation

## Description

- As a staff member, I want the system to seamlessly fall back to manual workflow when AI confidence drops below 80% and validate chronological plausibility of clinical event dates, so that data integrity is maintained even when AI is unreliable.

## Acceptance Criteria

1. **Given** AI confidence for a consolidation operation is below 80%, **When** the result is generated, **Then** the system presents the data in a manual review form pre-filled with AI suggestions marked as "low-confidence."
2. **Given** the system detects clinical event dates that violate chronological plausibility (e.g., procedure date before diagnosis date), **When** the validation runs, **Then** the conflicting dates are flagged with an explanation of the inconsistency.
3. **Given** a staff member is in manual fallback mode, **When** they edit and confirm data entries, **Then** each confirmed entry is saved with "manual-verified" status and staff attribution.
4. **Given** the AI service is completely unavailable, **When** a document is uploaded, **Then** the system displays a banner "AI unavailable — switch to manual" and provides the manual data entry form.

## Edge Cases

- What happens when a date field is partially parsed (e.g., only month/year)? System saves the partial date, flags it as "incomplete-date," and presents it for staff completion.
- How does the system handle conflicting timezone information in document dates? Phase 1 assumes all dates are in the clinic's local timezone; timezone metadata is ignored.
