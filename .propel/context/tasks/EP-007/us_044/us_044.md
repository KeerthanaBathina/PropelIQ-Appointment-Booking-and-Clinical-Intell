# User Story - US_044

## Story ID

- ID: US_044

## Story Title

- AI Conflict Detection Across Clinical Documents

## Description

- As a staff member, I want the AI to automatically detect conflicts across clinical documents (medication discrepancies, duplicate diagnoses, date inconsistencies) with critical conflicts escalated urgently, so that patient safety risks are flagged before clinical decisions are made.

## Acceptance Criteria

1. **Given** multiple documents are consolidated for a patient, **When** consolidation runs, **Then** the AI identifies medication discrepancies (different dosages, conflicting prescriptions) and flags them.
2. **Given** duplicate diagnoses with different dates are found, **When** the conflict is detected, **Then** the system flags the conflict with source citations from both documents.
3. **Given** a medication contraindication is detected, **When** the conflict is flagged, **Then** the system escalates it with an "URGENT" indicator and moves it to the top of the review queue.
4. **Given** AI confidence for conflict detection drops below 80%, **When** the result is generated, **Then** the system automatically flags the entire review for manual verification.
5. **Given** clinical event dates fail chronological plausibility validation, **When** the inconsistency is detected, **Then** the system flags the date conflict with a clear explanation.

## Edge Cases

- What happens when a conflict involves data from 3+ documents? System shows all conflicting sources in the comparison view, not just pairs.
- How does the system handle resolved conflicts when a new document is uploaded? Previously resolved conflicts are not reopened unless the new document introduces contradictory data.
