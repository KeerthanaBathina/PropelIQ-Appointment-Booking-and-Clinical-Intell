# User Story - US_079

## Story ID

- ID: US_079

## Story Title

- RAG Safety Controls

## Description

- As a safety engineer, I want prompt injection sanitization, access control on retrieved content, content filtering, and rate limiting (100 AI requests/user/hour) for the RAG pipeline, so that AI features are protected from abuse, unauthorized data access, and harmful outputs.

## Acceptance Criteria

1. **Given** user input is received for AI intake, **When** it is preprocessed, **Then** prompt injection patterns are detected and sanitized before being included in the AI prompt.
2. **Given** RAG retrieves clinical data chunks, **When** results are filtered, **Then** access control rules ensure the user can only view data from documents they are authorized to see.
3. **Given** AI generates a response, **When** content filtering runs, **Then** harmful, discriminatory, or medically dangerous content is blocked and replaced with a safe fallback message.
4. **Given** a user makes AI requests, **When** they exceed 100 requests per hour, **Then** the system returns HTTP 429 Too Many Requests with a Retry-After header.

## Edge Cases

- What happens when legitimate medical text triggers the prompt injection detector? System uses a medical terminology allowlist to reduce false positives from clinical terms.
- How does the system handle rate limiting for staff performing bulk operations? Staff accounts have a configurable higher rate limit (500/hour); admins can temporarily increase limits.
