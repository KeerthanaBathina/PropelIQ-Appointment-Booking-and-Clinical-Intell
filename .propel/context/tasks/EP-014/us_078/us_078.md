# User Story - US_078

## Story ID

- ID: US_078

## Story Title

- Hybrid Search and Knowledge Base Refresh

## Description

- As a developer, I want hybrid search combining semantic (pgvector) and keyword (PostgreSQL FTS) search with quarterly knowledge base refresh, so that retrieval accuracy is maximized and the knowledge base stays current with ICD-10/CPT updates.

## Acceptance Criteria

1. **Given** a search query is issued, **When** hybrid search runs, **Then** both pgvector cosine similarity and PostgreSQL full-text search are executed and results are merged with configurable weighting.
2. **Given** hybrid results are returned, **When** they are merged, **Then** duplicates are deduplicated and the combined relevance score determines final ranking.
3. **Given** a quarterly ICD-10/CPT code update is released, **When** the admin triggers a knowledge base refresh, **Then** new codes are chunked, embedded, and indexed while deprecated codes are marked.
4. **Given** separate vector indexes exist (medical terminology, intake templates, coding guidelines), **When** a search is scoped to a category, **Then** only the relevant index is queried.

## Edge Cases

- What happens when keyword search returns results that semantic search misses (exact code match)? Keyword results with exact matches are boosted to the top regardless of semantic score.
- How does the system handle a mid-refresh query? System serves queries from the existing index until refresh completes; atomic swap to new index on completion.
