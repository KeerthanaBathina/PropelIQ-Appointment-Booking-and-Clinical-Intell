# User Story - US_018

## Story ID

- ID: US_018

## Story Title

- Appointment Booking with Optimistic Locking

## Description

- As a patient, I want to book an appointment with guaranteed slot reservation, so that my selected time cannot be double-booked by another patient.

## Acceptance Criteria

1. **Given** I select an available slot, **When** I confirm the booking, **Then** the system creates the appointment using optimistic locking (version field compare-and-swap) and returns confirmation within 2 seconds.

2. **Given** another patient selects the same slot simultaneously, **When** both confirm, **Then** only the first transaction succeeds; the second receives "Slot no longer available" with refreshed availability.

3. **Given** I select a slot but do not confirm, **When** 1 minute passes, **Then** the hold is released and the slot returns to available inventory.

4. **Given** booking succeeds, **When** the confirmation is displayed, **Then** it shows appointment date, time, provider, and appointment type with a booking reference number.

## Edge Cases

- What happens when the database is temporarily unavailable during booking? System retries once, then displays "Service temporarily unavailable. Please try again."
- How does the system handle booking at the 90-day boundary? Slot at exactly 90 days is allowed; 91 days is rejected with date range error.
