# User Story - US_008

## Story ID

- ID: US_008

## Story Title

- Core Domain Entity Models & Migrations

## Description

- As a developer, I want EF Core 8 entity models for all 10 domain entities with code-first migrations, so that the database schema supports the full data model defined in the architecture design.

## Acceptance Criteria

1. **Given** entity models are defined, **When** a developer runs `dotnet ef migrations add CreateDomainEntities`, **Then** the migration generates DDL for Patient, Appointment, IntakeData, ClinicalDocument, ExtractedData, MedicalCode, User, AuditLog, QueueEntry, and NotificationLog.

2. **Given** the Appointment entity has a version field, **When** two concurrent updates target the same appointment, **Then** EF Core throws DbUpdateConcurrencyException for the second update (optimistic locking).

3. **Given** the Patient entity has a deleted_at column, **When** a patient is soft-deleted, **Then** global query filters exclude the record from standard queries while preserving it in the database.

4. **Given** JSONB columns are configured, **When** data is stored in preferred_slot_criteria, mandatory_fields, optional_fields, insurance_info, or data_content columns, **Then** EF Core serializes and deserializes JSON correctly.

5. **Given** enum mappings are configured, **When** an appointment status is set to "scheduled", **Then** the database stores the enum as a string value in the status column.

## Edge Cases

- What happens when a migration conflicts with existing data? Migrations include data migration scripts with rollback capability.
- How does the system handle JSONB column with invalid JSON? EF Core validation rejects the entity before persistence; error is logged.
